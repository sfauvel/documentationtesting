Description

This demo shows several ways of doing documentation for a same method.

Code comes from a RPG game that uses a hero. This hero has levels and gains experience to level up.

Note
 The examples shown here are generated from the source code. They therefore represent the behavior of the application at any times. Non regression is ensured by checking the absence of change in this document. Learn more here https://github.com/sfauvel/documentationtesting

View source of project on Github

Hero experience test

We show different approach to test writing a documentation. We start with a typical characterization test that produced a file using for non regression with approvals. After, we explore other way to produce a readable document. It gives ideas on advantages and drawbacks for each case.
